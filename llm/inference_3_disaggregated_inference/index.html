LLM Inference: Disaggregated Inference

December 15, 2023

What Is Disaggregated Inference (DI)? #

In LLM inference, the prefill and decode stages have different compute, memory, and bandwidth requirements. Prefill is typically compute-intensive, while decode is memory-bandwidth-intensive. Having both prefill and decode on the same node lead to interference. A prefill stage on a long prompt occupies the GPU and delays the decode stage for other requests. Hence, separating prefill and decode onto separate processes or hosts can improve performance. 1 2

Architecture #

The DI system includes a connector/scheduler that transfers the KV Cache between prefill nodes and decode nodes via NVLink/NVSwitch (intra-node) or GPUDirect RDMA (inter-node)
